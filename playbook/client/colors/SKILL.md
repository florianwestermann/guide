## Purpose
Define client color tokens and usage rules. Overrides general guidance when needed.

## Tokens (fill in)
- **Brand**: primary / secondary
- **Neutrals**: background / surface / text
- **Semantic**: success / warning / danger / info

## Usage rules
- Prefer semantic tokens in UI over raw hex values.
- Specify contrast targets for text and controls.
