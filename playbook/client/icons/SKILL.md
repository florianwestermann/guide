## Purpose
Define client iconography rules (style, sizing, usage). Overrides general guidance when needed.

## Tokens (fill in)
- **Style**: stroke/fill, corner radius, optical adjustments
- **Sizes**: e.g. 16/20/24 (and when to use each)
- **Stroke width**: per size

## Usage rules
- Prefer consistent visual weight across the set.
- Ensure icons have accessible names when interactive or meaningful.
